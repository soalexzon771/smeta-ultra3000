# 🔨 Смета-Ultra 3000

Одностраничное веб-приложение для подготовки сметы работ мастера по ремонту.

## Возможности

- **Гибкая админка** для настройки каталога работ:
  - единицы измерения;
  - категории работ;
  - работы с ценой, единицей измерения и формулой объёма;
  - шаблоны комнат с типовыми размерами.
- **Формирование сметы**:
  - добавление комнат с размерами (длина, ширина, высота);
  - автоматический расчёт площадей пола, потолка, стен и периметра;
  - выбор работ для каждой комнаты;
  - ручное или автоматическое задание количества работ;
  - итоговая сумма по смете.
- **Экспорт**:
  - в Excel (`.xlsx`) — через SheetJS;
  - в PDF — через jsPDF;
  - сохранение/загрузка сметы в JSON.
- **Сохранение конфигурации**:
  - выгрузка `config.json` с настройками админки;
  - загрузка `config.json` обратно;
  - сброс к настройкам по умолчанию.

## Запуск

Приложение работает без бэкенда и сборки. Достаточно открыть файл `index.html` в браузере.

### Быстрый старт через Python

```bash
cd smeta-ultra3000
python3 -m http.server 8765
```

Откройте в браузере: http://localhost:8765

### Или просто откройте файл

```bash
open index.html
```

## Использование

1. Откройте вкладку **Админка** и настройте:
   - единицы измерения;
   - категории и работы с ценами;
   - шаблоны комнат.
2. Скачайте `config.json`, чтобы сохранить настройки.
3. Перейдите во вкладку **Смета**:
   - укажите название объекта и дату;
   - добавьте комнаты и задайте размеры;
   - выберите работы для каждой комнаты;
   - при необходимости скорректируйте количество вручную.
4. Нажмите **XLSX** или **PDF** для экспорта сметы.

## Формулы объёма

Каждая работа может использовать одну из формул автоматического расчёта количества:

- **Площадь стен** — `2 * (длина + ширина) * высота`;
- **Площадь потолка** — `длина * ширина`;
- **Площадь пола** — `длина * ширина`;
- **Периметр** — `2 * (длина + ширина)`;
- **Фиксированное количество** — задаётся вручную (например, количество розеток).

## Единицы измерения и шаг

Шаг ввода количества работ настраивается у **единицы измерения**, а не у работы. Например:

- `м²`, `м.п.` — шаг `0.01`;
- `шт`, `комплект` — шаг `1`.

В админке, во вкладке **Единицы измерения**, можно задать шаг для каждой единицы.

## Пример config.json

```json
{
  "version": 1,
  "units": [
    { "name": "м²", "step": 0.01 },
    { "name": "м.п.", "step": 0.01 },
    { "name": "шт", "step": 1 },
    { "name": "комплект", "step": 1 }
  ],
  "categories": [
    {
      "id": "cat_xxx",
      "name": "Электрика",
      "works": [
        {
          "id": "work_yyy",
          "name": "Установка розетки",
          "unit": "шт",
          "price": 600,
          "formula": "fixed"
        }
      ]
    }
  ],
  "roomTemplates": [
    { "id": "tmpl_zzz", "name": "Кухня", "defaultLength": 3, "defaultWidth": 3, "defaultHeight": 2.7 }
  ]
}
```
